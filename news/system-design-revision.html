S2 SDv0.2 (Update)

Revised Data Flow Diagram incorporating feedback from the System Design Review. This update addresses both content corrections and format improvements to the Level 0 and Level 1 diagrams.

DateAuthorsDescription
Apr 17, 2026Zhiqi ZHANGDFD with content and format corrections

1. Corrections

The following changes address issues identified during the review phase:

LocationBeforeAfterReason
Right-side actor Patient Doctor The M2 Dashboard serves Medical Professionals, not patients. The right-side actor should reflect the actual user of M2
S2 → V1 data flow Formal data Format data More precisely describes the output: data that has been formatted into the 45-value CSV standard
V1 → V2 DB Recognition result Analysis results + Batch Format data V1 produces both real-time analysis results and batch-processed formatted data for storage
V2 DB ↔ M2 Operations for patient + Patient log Patient log + Doctor Feedback Added bidirectional Doctor Feedback flow between M2 and V2 DB to reflect the doctor's active role in prescribing and adjusting treatment
S2-01 ↔ S2-02 arrows Single bidirectional arrow Two unidirectional arrows Split into "Simulated raw IMU packet" (S2-01 → S2-02) and "Simulated data request" (S2-02 → S2-01), making each data flow independently traceable
