Team S2 Website is Live on GitHub Pages

The new Team S2 site is now published at rsdbkhusky.github.io/DSD2026_TeamS2 — a complete rebuild in pure HTML, CSS, and JavaScript, replacing the previous Vite-based deployment.

Date Author Description
Apr 1, 2026Zhiqi ZHANGWebsite launch announcement

The Team S2 website has been rebuilt from scratch and is now live on GitHub Pages. The previous site, built with Vite, has been replaced with a project that uses no build tools and no frameworks — plain HTML, CSS, and JavaScript, deployable directly from the repository's main branch without any CI pipeline.

Four Pages

The site is structured around four dedicated pages, each serving a distinct purpose:

  • Home — project overview, system architecture, team responsibilities, and workflow
  • Progress — sprint-by-sprint status, starting with Sprint 1 tasks
  • News — team announcements and milestones, with individual article pages
  • Member — individual profiles for all five S2 members, plus the role structure table

Technical Choices

The site intentionally avoids build complexity. Every page is a standalone HTML file that shares one stylesheet (style.css) and one script (script.js). Adding a new page or modifying content requires no toolchain — just edit the file and push.

Visual effects — the cursor glow, scroll progress bar, card shimmer lines, and reveal animations — are each implemented as self-contained, independently removable pieces. Deleting a single element or class from the HTML disables any given effect without touching anything else.

What the Site Will Track

The Progress and News pages will be updated as Sprint 1 work advances. The next entries will cover the formation of the team's working structure and — once it begins — the IF1 contract definition process with Team V1.
