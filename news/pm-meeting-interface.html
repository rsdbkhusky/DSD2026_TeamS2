PM Meeting & Interface Discussion

A cross-team PM meeting on April 18 aligned the current progress across all teams, followed by private interface discussions between S2 and its neighbouring teams (S1 and V1) to finalise the data exchange details.

DateAuthorsDescription
Apr 20, 2026Zhiqi ZHANG, João NevesPM meeting summary and interface discussion outcomes

1. Cross-Team PM Meeting

On the evening of April 18, 2026, a cross-team PM meeting was held. Zhiqi ZHANG attended as the S2 team leader. The meeting reviewed the current progress of each team, aligned ongoing work across the six groups, and discussed the overall project timeline for the upcoming sprints.

A key focus of the meeting was the inter-team data interface alignment. Each team clarified the data they need to send and receive across team boundaries, identified the information that must be exchanged through IF1 and IF2, and flagged open questions that still required further discussion. Teams were encouraged to follow up with private bilateral discussions after the meeting to resolve remaining details.

2. Interface Discussion with S1 (Sensor Firmware)

Zhiqi ZHANG conducted a private discussion with the S1 team to clarify the detailed interface between S1 and S2. The outcome:

  • S1 will be responsible for raw sensor data acquisition from the physical IMU hardware.
  • The existing sensor software provides an SDK for developers, which is capable of capturing IMU orientation angles (attitude/heading).
  • The data that S1 will pass to S2 is the IMU orientation angle readings.
  • Development and data collection on the S1 side are currently in progress.

3. Interface Discussion with V1 (AI Engine)

João Neves reached out to the V1 team to clarify whether they require any data from S2 beyond the target angle (the interface item confirmed during the PM meeting). V1 confirmed that, in addition to the target angle, they ideally need the following data from S2:

Data ItemDescription
Measured sensor data / motion streamThe real-time IMU data stream produced by S2 (45-value CSV samples)
Timestamp or sequence informationTime or ordering metadata for each sample to support temporal analysis
Sensor / joint identificationMapping of data fields to specific sensor nodes or body joints
Format and unitsAs defined in the IF1 contract (e.g. units for angles, coordinate frame conventions)
Payload status / error informationIndicators of data validity, sensor connectivity, or missing readings
Target angle / expected valuesReference values used by V1 for motion comparison and accuracy scoring

After analysis, S2 concluded that all of the above items can be provided by S2, with one exception: the target angle expected values. These reference values represent the prescribed exercise targets set by Medical Professionals, which are stored and managed by V2 (Backend API & Storage). S2's role is to provide real-time measured data, not the prescription data that originates from the clinical side.

This conclusion was communicated to the V1 team leader, who confirmed the assessment. V1 will obtain the target angle expected values from V2 directly. The discussion was productive and both sides expressed satisfaction with the outcome.
