IUC-S2-02-01 Deliver Formal Sensor Data

ElementDescription
ReferenceIUC-S2-02-01
ActorsM1, S2, V2
GoalAllow M1 AppFrontend to obtain formal sensor data, while simultaneously transmitting it to the server.
SummaryM1 sends a data delivery request to S2. S2 returns the data accumulated since the last delivery to M1 via the return value, and simultaneously transmits it to V2 over the network using a Token for authentication.
TriggerPeriodically called by M1's internal logic.
PreconditionThe session's data processing work has already started.
PostconditionsNone

Basic Flow

StepM1S2V2
1M1 calls S2's data delivery method.
2Packages the data accumulated since the last delivery into Formal Sensor Data.
3Transmits the Formal Sensor Data over the network, along with the user Token and session meta data for authentication and identification.
4Starts a subprocess to receive V2's confirmation (only one needs to be started per session).
5Returns the Formal Sensor Data.
6Receives the Formal Sensor Data and proceeds with subsequent steps.
7Receives the Formal Sensor Data, user Token, and session meta data, and performs meta data verification.
8Returns a confirmation signal.
9Receives the confirmation signal; the subprocess continues.

Alternative Flow

Occurrence StepConditionSystem Response
3Network transmission fails.S2 method is interrupted and returns an error. M1 displays an error message on screen. Since the data has not been synced to the database, the step may be terminated.
4Subprocess fails to start.S2 method is interrupted and returns an error. M1 displays an error message on screen. Since the data may not have been synced to the database, the step may be terminated.
6S2 method does not finish normally / times out.M1 displays an error/timeout message on screen. The step is terminated.
7V2 verification fails.V2 sends an error message. S2's subprocess receives it and interrupts, returning an error to the main method. The main method returns the error on the next call of this use case by M1. M1 displays an error message on screen. Since the data has not been synced to the database, the step may be terminated.
9V2 does not respond for an extended period.S2's subprocess interrupts and returns an error to the main method. The main method returns the error on the next call of this use case by M1. M1 displays an error message on screen. Since the data may not have been synced to the database, the step may be terminated.
